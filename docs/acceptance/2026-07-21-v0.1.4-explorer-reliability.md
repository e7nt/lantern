# v0.1.4 explorer reliability — 2026-07-21

## User outcome

Selecting any visible file in the workbench explorer opens it in the existing
Helix process on supported Apple Silicon Macs. An idle explorer remains visually
still instead of clearing and repainting every 750 ms.

## Root causes and corrections

The first explorer release used GNU-only `realpath -m` and `--relative-to`
flags in a helper shipped exclusively for macOS. The helper now canonicalizes
only existing repository files with portable `realpath` invocation and derives
the already-validated relative path through shell parameter expansion.

The background Git refresh previously marked every successful snapshot as
dirty even when the tree, status, comments, and expansion state were unchanged.
Snapshots now compare after preserving expansion state and request a repaint
only for a visible change. Errors likewise repaint only when their message
changes.

## Regression boundary

- Terminal contracts run navigation with a deliberately BSD-like `realpath`
  wrapper that rejects option flags.
- Explorer tests distinguish identical and changed background snapshots.
- Homebrew acceptance no longer stops after pane startup: it selects the real
  committed `README.md` in Explorer, requires focus to move to Helix, and
  verifies that the shipped file content is visible in the editor.
- The complete canonical project gate remains required before publication.
