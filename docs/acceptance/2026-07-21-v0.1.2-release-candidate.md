# v0.1.2 release candidate — 2026-07-21

## Release outcome

`v0.1.2` is a bounded developer-preview release. It adds the persistent,
Git-aware workbench explorer and removes balanced inline Markdown chrome from
agent answers at the terminal presentation boundary. It also makes Homebrew
acceptance compare the installed binary with the formula version instead of a
hard-coded prior release.

This checkpoint does not claim completion of Lantern's full product roadmap.
Guided Build playback, structured learning missions and change narratives,
multi-folder workbenches, Live Collaboration, and Linux release artifacts
remain outside this patch release. The implemented single-repository developer
preview is the release boundary.

## Verification

- The canonical `./scripts/check.sh` gate passes at version `0.1.2`.
- Every Rust workspace package, both Python projects, the private Node runtime
  package, their lock files, and the repository `VERSION` agree on `0.1.2`.
- A fresh detached workbench reports `lantern 0.1.2` and starts live Explorer,
  Helix, and Lantern panes at the intended 20/80 upper split and full-width 20%
  agent layout.
- The subscription-backed Protocol v17 live trace passes grounded admission,
  a 2.21-second warm follow-up, and active interruption.
- The subscription-backed external-edit journey changes only its intended
  implementation and test files, passes the focused test, leaves both files
  unstaged for review, and interrupts the separate active inspection in 11 ms.
- Deterministic terminal tests prove headings, lists, fenced code, emphasis,
  and inline code render without Markdown delimiter noise while incomplete
  streamed delimiters and ordinary source characters remain intact.
- Tracked files and full Git history contain no credential-shaped values;
  GitHub secret scanning, push protection, vulnerability alerts, and automated
  dependency security fixes are enabled, with zero open secret alerts.

## Remaining release evidence

The tag workflow must still build, checksum, attest, and publish both macOS
archives and update the tap. After publication, the Homebrew acceptance workflow
must install and launch the released formula on both architectures. A true
`v0.1.1` to `v0.1.2` Cellar upgrade remains an explicit hardening follow-up and
is not represented as completed by a fresh install.
